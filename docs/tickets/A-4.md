## 1. What Was Delivered

A-4 adds a regex-based heuristic chapter-detection fallback for PDFs
that have no outline/bookmark tree, and wires it into the existing
`--show-chapters` flag from A-3.

**Source files:**
- `src/gbc_reader_prep/chapters.py` (modified) — adds:
  - `detect_chapters_from_heuristic(doc) -> list[Chapter]` — scans each
    page's extracted text (`page.get_text()`) line by line for the
    first line matching one of `_HEURISTIC_PATTERNS`: `^chapter\s+\d+`,
    `^prologue\b`, `^epilogue\b`, `^introduction\b` (all
    case-insensitive, anchored to line start). Each match becomes a
    `Chapter` at `level=1`.
  - `detect_chapters_from_heuristic_path(pdf_path)` — path wrapper.
  - `detect_chapters(doc)` / `detect_chapters_path(pdf_path)` — try the
    outline first, fall back to the heuristic if the outline is empty.
- `src/gbc_reader_prep/preprocess.py` (modified) — `--show-chapters`
  now calls `detect_chapters_path` (the combined entry point) instead
  of the outline-only `detect_chapters_from_outline_path`, so PDFs
  without bookmarks now produce a chapter list too. Updated the
  no-chapters log message and the flag's `--help` text accordingly.

**Tests:**
- `tests/test_chapters.py` (modified) — adds a `_make_pdf_with_text_pages`
  fixture helper (uses `page.insert_text`, no `set_toc`) and 9 new tests
  covering heuristic matching (`Chapter N`, `Prologue`/`Epilogue`,
  case-insensitivity, no-match, missing file, direct `Document` entry
  point) plus 3 tests for the combined `detect_chapters`/
  `detect_chapters_path` fallback behavior. 24/24 tests pass.

## 2. Design Notes

- **Why regex on `page.get_text()` lines, not the whole page string:**
  Splitting into lines and anchoring with `^` keeps the patterns simple
  and avoids false positives from "Chapter 1" appearing mid-sentence
  (e.g. in a table of contents page) — only a line that *starts* with
  the pattern counts. This is a known limitation: a PDF's own ToC page
  (rendered as text, not a real outline) will likely produce false
  matches for every entry on that page. A-5 (front/back matter
  trimming) is the natural place to filter these out if it becomes a
  problem in practice.
- **Why all heuristic chapters are `level=1`:** there is no general way
  to infer outline depth from page text alone; the outline-based path
  remains the only source of `level > 1` chapters.
- **Why `detect_chapters` lives in the same module:** keeps the
  library/CLI layering rule from A-2 §4.1 — `preprocess.py` needed a
  single call that "just works" regardless of whether the PDF has an
  outline, without itself encoding the fallback logic.
- **Pattern list (`_HEURISTIC_PATTERNS`) is a module-private constant.**
  Not part of the public API; if future tickets need to extend the
  pattern set, edit this list directly rather than exposing it as a
  parameter, unless a real use case demands configurability.
