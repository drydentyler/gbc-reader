## 1. What Was Delivered

A-5 adds back-matter detection and a CLI dry-run (`--inspect`) for
proposed main-content page bounds, plus `--start-page`/`--end-page`
overrides.

**Source files:**

- `src/gbc_reader_prep/trim.py` (new) —
  - `TrimResult` — frozen dataclass: `start_page`, `end_page`
    (inclusive), `back_matter_title`, `back_matter_start_page`.
  - `detect_back_matter_chapter(chapters)` — finds the earliest chapter
    whose title matches one of `Appendix`, `Notes`, `Bibliography`,
    `Index`, `About the Author`, `Acknowledgments`/`Acknowledgements`
    (case-insensitive, anchored).
  - `detect_content_bounds(chapters, page_count)` — proposes
    `start_page` (first chapter's start, or `0`) and `end_page`
    (`page_count - 1`, or the page before detected back matter). Ignores
    a back-matter match at or before the proposed start page so the
    whole book is never trimmed away by a false positive on the first
    chapter.
- `src/gbc_reader_prep/preprocess.py` (modified) —
  - `--inspect` flag: dry run. Opens the PDF, runs `detect_chapters_path`
    and `detect_content_bounds`, applies any `--start-page`/`--end-page`
    overrides, logs a report (proposed range, detected back matter,
    chapter list), and returns without writing any output file.
  - `--start-page` / `--end-page` (`int`, default `None`): override the
    detected bounds. Usable with or without `--inspect`.
  - `-o`/`--output` is now optional at the argparse level; `run()`
    enforces it is required unless `--inspect` is given (exit code `2`
    otherwise).

**Tests:**

- `tests/test_trim.py` (new) — 11 tests covering pattern matching for
  each back-matter title, earliest-match selection, no-match/empty-input
  cases, bounds computation with and without back matter, the
  first-chapter-false-positive guard, and the `page_count < 1` error.
- `tests/test_cli.py` (modified) — 5 new tests: `--output` required
  without `--inspect`, `--inspect` reports proposed bounds and writes no
  file, `--inspect` on a missing PDF returns `2`, and
  `--start-page`/`--end-page` overrides are reflected in the report.

## 2. Design Notes

- **Back-matter only, not front-matter trimming of pages before chapter
  1:** the project plan's "main content start" heuristic is already
  satisfied by A-3/A-4's chapter detection (the first detected chapter's
  `start_page`). A-5 focuses on the back-matter half flagged as the open
  problem in the A-4 synopsis.
- **Why ignore a back-matter match at/before the proposed start page:**
  protects against the heuristic fallback's known false-positive risk
  (e.g. a rendered table-of-contents page whose lines start with
  `Index` or similar) trimming away the entire book.
- **`--output` becomes optional, enforced at runtime:** keeps the
  argparse-level contract simple (no mutually-exclusive-group
  complexity) while still failing fast with exit code `2` if a caller
  forgets `--output` outside of `--inspect`.

## 3. Follow-ups for Later Tickets

- `--inspect` currently re-opens the PDF via `pymupdf` directly inside
  `preprocess.py` to get `page_count`; if a third caller needs
  page-count-only access, consider exposing a small helper in
  `extract.py` instead of duplicating the `pymupdf.open()`/`close()`
  pattern.
- A-6 onward (cover image, pagination, `.book` writer) should consume
  `detect_content_bounds`'s `start_page`/`end_page` (or CLI overrides)
  to actually trim pages from the pipeline, not just report them.
