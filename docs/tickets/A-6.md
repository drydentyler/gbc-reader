## 1. What Was Delivered

A-6 adds cover image extraction: the first page of the PDF is rendered,
downscaled to the display resolution (400x240), and 1-bit
Floyd-Steinberg dithered, then made available both as a saved PNG and as
base64 for later embedding in the `.book` file (A-8).

**Source files:**

- `src/gbc_reader_prep/cover.py` (new) —
  - `render_cover(pdf_path)` — opens the PDF, renders page 0 via PyMuPDF
    at a 2x zoom (so downscaling has real detail to dither against),
    converts to grayscale, resizes to 400x240, and converts to mode `"1"`
    (Pillow's default 1-bit conversion applies Floyd-Steinberg
    dithering).
  - `cover_to_base64(image)` — encodes a Pillow Image as a base64 PNG
    string.
  - `extract_cover(pdf_path, out_dir)` — saves the rendered cover as
    `cover.png` in `out_dir` and returns `(cover_path, base64_string)`.
- `src/gbc_reader_prep/preprocess.py` (modified) —
  - `--extract-cover` flag: after text extraction, renders and saves the
    cover image alongside `--output` (in the same directory).
- `pyproject.toml` (modified) — added `pillow>=10.0.0` dependency.

**Tests:**

- `tests/test_cover.py` (new) — 6 tests covering: 1-bit mode and
  400x240 size of the rendered image, `FileNotFoundError` for a missing
  PDF, base64 round-trips to a valid PNG, `extract_cover` saves the file
  and returns matching base64, and CLI integration via `--extract-cover`
  (success and missing-PDF cases).

## 2. Design Notes

- **Render at 2x zoom before downscaling:** rendering directly at
  400x240 via PyMuPDF's pixmap matrix would alias text-heavy cover
  pages; rendering larger and downscaling with Lanczos resampling first
  gives the subsequent dither something with real gradients to work
  with.
- **PNG, not raw bitmap:** `.book` files are JSON, so the cover needs to
  be a self-describing, easily-decodable format for both the base64
  embed and the on-disk inspection copy. PNG is small for 1-bit images
  and Pillow handles it natively.
- **`extract_cover` returns both a path and a base64 string** rather
  than just one or the other, since the issue asks for both an on-disk
  copy (for the CLI's output directory) and the base64 (for the future
  `.book` writer in A-8) from a single render pass.
