## 1. What Was Delivered

A-8 assembles the manifest, chapters, pages, and cover produced by
earlier tickets (A-3/A-4 chapter detection, A-5 trimming, A-6 cover
extraction, A-7 pagination) into a single JSON `.book` file, per the
schema in the project plan (§3.1).

**Source files:**

- `src/gbc_reader_prep/book.py` (new) —
  - `SCHEMA_VERSION = 1`
  - `build_book(pdf_path, *, title=None, author=None, font_metrics=DEFAULT_FONT_METRICS, start_page=None, end_page=None, include_cover=True) -> dict`
    — assembles the full manifest dict in memory: `schema_version`,
    `title`/`author` (from PDF metadata, falling back to the filename
    stem), `display` (`width_px`/`height_px`/`font`), `cover_png_base64`
    (rendered directly via `cover.render_cover`/`cover_to_base64`, not
    `extract_cover`, so no stray `cover.png` is written next to the
    source PDF), `chapters` (`id`/`title`/`start_page`, matching the
    chapter subset `paginate_chapters` actually laid out), `pages`
    (`chapter_id`/`lines`), and `total_pages`.
  - `write_book_path(pdf_path, out_path, **build_kwargs) -> dict` —
    builds the manifest and writes it as JSON to `out_path`, creating
    parent directories as needed.
- `src/gbc_reader_prep/preprocess.py` (modified) —
  - `run()` now dispatches to a new `_write_book(args)` handler when
    `--output`'s suffix is `.book` (instead of the plain `.txt`
    extraction path). `--start-page`/`--end-page`/`--font-metrics` are
    honored on this path; `--extract-cover`/`--show-chapters`/
    `--paginate` remain scoped to the legacy `.txt`-extraction
    diagnostics path, since a `.book` file always includes all of
    chapters/cover/pages regardless of those flags.

**Tests:**

- `tests/test_book.py` (new, 14 tests) — covers `build_book`'s schema
  fields, title/author metadata fallback, chapter/page id consistency,
  cover base64 round-trip, `include_cover=False`, start/end page
  overrides, missing-PDF `FileNotFoundError`, `write_book_path`'s JSON
  round-trip, and CLI integration (`preprocess -o *.book`: success,
  missing PDF, missing `--font-metrics` file, page overrides).

## 2. Acceptance Criteria

- **File opens successfully.** Verified manually: ran
  `gbc-reader-prep preprocess sample.pdf -o sample.book` against a
  synthetic fixture PDF and `json.load()`'d the result successfully.
- **Schema validates correctly.** `build_book`'s output dict shape is
  asserted field-by-field in `test_book.py`; matches the project plan's
  `.book` schema example in §3.1.
- **File contains expected page count, chapter count, and cover
  image.** `total_pages == len(pages)`, `chapters` matches detected
  chapters (id/title/start_page), and `cover_png_base64` decodes to a
  valid PNG (magic-byte checked) — all asserted in `test_book.py` and
  manually re-verified against the sample fixture.

## 3. Design Notes

- **Cover rendering bypasses `extract_cover`.** `extract_cover` both
  renders the cover *and* saves a `cover.png` file as a side effect.
  `build_book` only wants the base64 string, and doesn't want to litter
  the source PDF's directory with an unrelated PNG every time a `.book`
  file is built — so it calls `render_cover`/`cover_to_base64` directly.
- **Dispatch by `--output` suffix, not a new flag.** The project plan's
  CLI design (§3.1) shows `--output mybook.book` as the natural
  end-state of `preprocess`, with no separate flag. Existing `.txt`-based
  tests/flags (`--extract-cover`, `--show-chapters`, `--paginate`)
  continue to work unchanged for non-`.book` output paths, so this is a
  non-breaking, purely additive change.
- **Chapter id/title/start_page must match `pages[].chapter_id`
  exactly.** `paginate_chapters` builds its own in-range,
  `start_page`-sorted chapter subset internally (with a blank-title
  single-chapter fallback when none fall in range) and that's what
  `Page.chapter_id` indexes into — not the raw, unfiltered chapter list.
  `book._relevant_chapters` duplicates that exact subset-selection logic
  (mirroring the same pattern already used by
  `preprocess._report_pagination`/`_write_paginate_output`) so the
  manifest's `chapters` array lines up with `pages[].chapter_id`.
